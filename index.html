1. Python Basics

Comments - Your Code's Documentation

Comments are essential for writing maintainable code. They explain what your code does, why it does it, and how it works. Think of comments as notes to your future self and other developers. Good commenting is a hallmark of professional programming.

üí° Why Comments Matter:

  • Code Clarity: Explain complex logic and algorithms
  • Maintenance: Help others (and future you) understand the code
  • Documentation: Describe function purposes and parameters
  • Debugging: Temporarily disable code without deleting it
# Single line comment - use # for quick notes

'''
Multi-line comments
can span multiple lines
and are great for documentation
'''

# Comments are ignored by Python interpreter
print("Hello World")  # This comment explains the code

Print Statements - Your Window to Output

The print() function is your primary tool for displaying information to users and debugging your programs. It's the most fundamental way to see what your code is doing and communicate results. Understanding print() thoroughly will make you a more effective programmer.

üîç Print Function Features:

  • Multiple Arguments: Print several values at once
  • Separators: Control how values are separated
  • End Characters: Control what comes after the output
  • File Output: Direct output to files instead of console
# Basic print statements
print("Hello World")        # String output
print(42)                   # Integer output
print(3.14)                 # Float output
print(9 + 9)               # Expression result: 18

# Print multiple items with automatic spacing
print("Age:", 25)           # Output: Age: 25
print("Python", "is", "awesome")  # Output: Python is awesome

# Advanced print features
print("Name", "Age", "City", sep=" | ")  # Custom separator
print("Loading", end="...")              # Custom end character
print("Done!")                           # Continues on same line

# Print with formatting
name = "Alice"
score = 95.5
print(f"Student: {name}, Score: {score}%")  # f-string formatting

# Print for debugging
values = [1, 2, 3, 4, 5]
print(f"Debug - List contents: {values}")
print(f"Debug - List length: {len(values)}")

Escape Sequences - Formatting Your Text

Escape sequences are special character combinations that represent characters that are difficult or impossible to type directly. They give you precise control over text formatting and are essential for creating professional-looking output.

üìù Common Escape Sequences:

  • \n: New line (line break)
  • \t: Tab character (indentation)
  • \": Double quote within strings
  • \': Single quote within strings
  • \\: Backslash character
  • \r: Carriage return
# New line formatting
print("First line\nSecond line\nThird line")

# Tab alignment for clean output
print("Name:\tJohn Doe")
print("Age:\t25")
print("City:\tNew York")
print("Occupation:\tDeveloper")

# Multi-line strings with triple quotes
poem = """Roses are red,
Violets are blue,
Python is awesome,
And so are you!"""
print(poem)

# Quotes within strings
print("She said, \"Python is my favorite language!\"")
print('He replied, \'I agree completely!\'')

# File paths (Windows style)
print("File location: C:\\Users\\Python\\Projects\\main.py")

# Creating formatted tables
print("Product\t\tPrice\t\tStock")
print("Laptop\t\t$999\t\t15")
print("Mouse\t\t$25\t\t50")
print("Keyboard\t$75\t\t30")

# Raw strings (ignore escape sequences)
print(r"This is a raw string: \n \t \\")

2. Variables and Data Types

Variable Basics - Storing and Managing Data

Variables are the foundation of programming. They're named containers that store data values, allowing you to save, modify, and reuse information throughout your program. Python's dynamic typing system automatically determines the data type, making it beginner-friendly while remaining powerful.

üéØ Variable Best Practices:

  • Descriptive Names: Use clear, meaningful variable names
  • Snake Case: Use underscores for multi-word names (user_name)
  • Avoid Keywords: Don't use Python reserved words
  • Case Sensitivity: Remember that 'Name' and 'name' are different
  • Constants: Use ALL_CAPS for values that shouldn't change
# Creating variables with different data types
name = "Guido Van Rossum"    # String - text data
age = 67                     # Integer - whole numbers
height = 5.9                 # Float - decimal numbers
is_programmer = True         # Boolean - True/False values
salary = None               # None - represents absence of value

# Check data types
print(f"Name: {name} (Type: {type(name).__name__})")
print(f"Age: {age} (Type: {type(age).__name__})")
print(f"Height: {height} (Type: {type(height).__name__})")
print(f"Is Programmer: {is_programmer} (Type: {type(is_programmer).__name__})")

# Variable reassignment (dynamic typing)
user_id = 12345        # Initially an integer
print(f"User ID: {user_id} (Type: {type(user_id).__name__})")

user_id = "USER_12345"  # Now it's a string
print(f"User ID: {user_id} (Type: {type(user_id).__name__})")

# Multiple assignment
x, y, z = 10, 20, 30
print(f"x={x}, y={y}, z={z}")

# Constants (by convention, use ALL_CAPS)
PI = 3.14159
MAX_USERS = 1000
DEFAULT_COLOR = "blue"

# Variable naming examples
user_name = "alice"          # Good: descriptive, snake_case
total_score = 95            # Good: clear purpose
is_valid = True             # Good: boolean naming
# userName = "bob"          # Avoid: camelCase in Python
# x = "data"                # Avoid: non-descriptive names

Lists - Dynamic Ordered Collections

Lists are one of Python's most versatile data structures. They store multiple items in a specific order and can be modified after creation. Lists can contain different data types and grow or shrink as needed, making them perfect for managing collections of related data.

üîß List Capabilities:

  • Ordered: Items maintain their position
  • Mutable: Can be changed after creation
  • Duplicates Allowed: Same values can appear multiple times
  • Mixed Types: Can store different data types together
  • Indexing: Access items by position (starting at 0)
  • Slicing: Extract portions of the list
# Creating different types of lists
fruits = ['apple', 'banana', 'orange', 'grape']
numbers = [1, 2, 3, 4, 5]
mixed_data = ['Alice', 25, True, 3.14, 'Engineer']
empty_list = []  # Empty list to fill later

# Accessing elements with indexing
print(f"First fruit: {fruits[0]}")      # apple
print(f"Last fruit: {fruits[-1]}")      # grape
print(f"Second fruit: {fruits[1]}")     # banana

# List slicing (extracting portions)
print(f"First 3 fruits: {fruits[0:3]}")     # ['apple', 'banana', 'orange']
print(f"Last 2 fruits: {fruits[-2:]}")      # ['orange', 'grape']
print(f"Every other fruit: {fruits[::2]}")  # ['apple', 'orange']

# Modifying lists
fruits[0] = 'mango'           # Change first item
fruits.append('kiwi')         # Add to end
fruits.insert(1, 'pineapple') # Insert at specific position
print(f"Modified fruits: {fruits}")

# List methods for manipulation
shopping_list = ['bread', 'milk', 'eggs']
shopping_list.extend(['cheese', 'butter'])  # Add multiple items
shopping_list.remove('milk')                # Remove specific item
popped_item = shopping_list.pop()           # Remove and return last item
print(f"Shopping list: {shopping_list}")
print(f"Removed item: {popped_item}")

# List information and searching
scores = [85, 92, 78, 96, 88, 92]
print(f"List length: {len(scores)}")
print(f"Highest score: {max(scores)}")
print(f"Lowest score: {min(scores)}")
print(f"Average score: {sum(scores) / len(scores):.1f}")
print(f"Count of 92: {scores.count(92)}")
print(f"Index of 96: {scores.index(96)}")

# List comprehension (advanced)
squared_numbers = [x**2 for x in range(1, 6)]
print(f"Squared numbers: {squared_numbers}")  # [1, 4, 9, 16, 25]

Tuples - Immutable Ordered Collections

Tuples are ordered collections that cannot be changed after creation. This immutability makes them perfect for storing data that shouldn't be modified, like coordinates, database records, or configuration settings. They're faster than lists and can be used as dictionary keys.

üîí Tuple Advantages:

  • Immutable: Cannot be changed, ensuring data integrity
  • Faster: More efficient than lists for read-only data
  • Hashable: Can be used as dictionary keys
  • Unpacking: Easy assignment to multiple variables
  • Return Multiple Values: Functions can return tuples
# Creating tuples
coordinates = (10, 20)
person_info = ('Alice', 28, 'Software Engineer', 'New York')
rgb_colors = ('red', 'green', 'blue')
single_item = (42,)  # Note the comma for single-item tuples
empty_tuple = ()

# Accessing elements
print(f"X coordinate: {coordinates[0]}")
print(f"Y coordinate: {coordinates[1]}")
print(f"Person name: {person_info[0]}")
print(f"Person age: {person_info[1]}")

# Tuple slicing (like lists)
print(f"First two person details: {person_info[:2]}")
print(f"Last two details: {person_info[-2:]}")

# Tuple unpacking (very useful!)
x, y = coordinates
name, age, job, city = person_info
print(f"Coordinates: X={x}, Y={y}")
print(f"Person: {name}, {age} years old, works as {job} in {city}")

# Partial unpacking with *
first, *middle, last = (1, 2, 3, 4, 5)
print(f"First: {first}, Middle: {middle}, Last: {last}")

# Tuples are immutable - this would cause an error:
# coordinates[0] = 15  # TypeError: 'tuple' object does not support item assignment

# Tuple methods
numbers = (1, 2, 3, 2, 4, 2, 5)
print(f"Count of 2: {numbers.count(2)}")
print(f"Index of 4: {numbers.index(4)}")
print(f"Tuple length: {len(numbers)}")

# Practical example: Function returning multiple values
def get_name_age():
    return "Bob", 30  # Returns a tuple

user_name, user_age = get_name_age()
print(f"User: {user_name}, Age: {user_age}")

# Using tuples as dictionary keys
locations = {
    (0, 0): "Origin",
    (1, 2): "Point A",
    (-1, -1): "Point B"
}
print(f"Location at (1, 2): {locations[(1, 2)]}")

Sets - Unique Value Collections

Sets are unordered collections of unique elements. They automatically eliminate duplicates and provide powerful operations for comparing and combining collections. Sets are perfect for membership testing, removing duplicates, and mathematical set operations like unions and intersections.

‚ö° Set Operations:

  • Unique Elements: Automatically removes duplicates
  • Fast Membership Testing: Quickly check if item exists
  • Mathematical Operations: Union, intersection, difference
  • Mutable: Can add/remove elements after creation
  • Unordered: No guaranteed order of elements
# Creating sets
unique_numbers = {1, 2, 3, 3, 4, 4, 5}  # Duplicates automatically removed
print(f"Unique numbers: {unique_numbers}")  # {1, 2, 3, 4, 5}

# Creating sets from lists (removes duplicates)
duplicates_list = [1, 2, 2, 3, 3, 3, 4]
unique_from_list = set(duplicates_list)
print(f"Original list: {duplicates_list}")
print(f"Unique values: {unique_from_list}")

# Set operations
students_math = {'Alice', 'Bob', 'Charlie', 'Diana'}
students_science = {'Bob', 'Diana', 'Eve', 'Frank'}

# Union (all students in either class)
all_students = students_math.union(students_science)
print(f"All students: {all_students}")

# Intersection (students in both classes)
both_classes = students_math.intersection(students_science)
print(f"Students in both classes: {both_classes}")

# Difference (students only in math)
only_math = students_math.difference(students_science)
print(f"Only in math class: {only_math}")

# Symmetric difference (students in exactly one class)
exactly_one_class = students_math.symmetric_difference(students_science)
print(f"In exactly one class: {exactly_one_class}")

# Set methods
fruits = {'apple', 'banana', 'orange'}
fruits.add('grape')           # Add single element
fruits.update(['kiwi', 'mango'])  # Add multiple elements
print(f"Updated fruits: {fruits}")

fruits.remove('banana')       # Remove element (raises error if not found)
fruits.discard('pineapple')   # Remove element (no error if not found)
print(f"After removal: {fruits}")

# Membership testing (very fast)
print(f"Is 'apple' in fruits? {'apple' in fruits}")
print(f"Is 'banana' in fruits? {'banana' in fruits}")

# Practical example: Finding common interests
alice_hobbies = {'reading', 'swimming', 'coding', 'music'}
bob_hobbies = {'gaming', 'coding', 'music', 'sports'}

common_hobbies = alice_hobbies & bob_hobbies  # Shorthand for intersection
print(f"Alice and Bob both enjoy: {common_hobbies}")

# Set comprehension
squared_evens = {x**2 for x in range(10) if x % 2 == 0}
print(f"Squared even numbers: {squared_evens}")

Dictionaries - Key-Value Data Storage

Dictionaries are Python's most flexible data structure, storing data as key-value pairs. They're like real-world dictionaries where you look up a word (key) to find its definition (value). Dictionaries are perfect for representing structured data, configurations, and creating fast lookups.

üóùÔ∏è Dictionary Features:

  • Key-Value Pairs: Associate keys with values for fast lookup
  • Mutable: Can be modified after creation
  • Unique Keys: Each key can appear only once
  • Any Data Type: Values can be any Python object
  • Fast Access: O(1) average time complexity for lookups
  • Ordered: Maintain insertion order (Python 3.7+)
# Creating dictionaries
student = {
    "name": "Alice Johnson",
    "age": 20,
    "grade": "A",
    "subjects": ["Math", "Physics", "Computer Science"],
    "is_enrolled": True,
    "gpa": 3.8
}

# Accessing values
print(f"Student name: {student['name']}")
print(f"Student age: {student.get('age')}")
print(f"Student phone: {student.get('phone', 'Not provided')}")  # Default value

# Modifying dictionaries
student["age"] = 21                    # Update existing value
student["email"] = "alice@email.com"   # Add new key-value pair
student["subjects"].append("Chemistry") # Modify nested list

print(f"Updated student info: {student}")

# Dictionary methods
print(f"All keys: {list(student.keys())}")
print(f"All values: {list(student.values())}")
print(f"All items: {list(student.items())}")

# Iterating through dictionaries
print("\nStudent Information:")
for key, value in student.items():
    print(f"{key.title()}: {value}")

# Nested dictionaries (complex data structures)
company = {
    "name": "Tech Corp",
    "employees": {
        "john": {"position": "Developer", "salary": 75000},
        "sarah": {"position": "Designer", "salary": 65000},
        "mike": {"position": "Manager", "salary": 85000}
    },
    "departments": ["Engineering", "Design", "Marketing"]
}

print(f"\nJohn's salary: ${company['employees']['john']['salary']:,}")
print(f"Company departments: {', '.join(company['departments'])}")

# Dictionary methods for manipulation
scores = {"Alice": 95, "Bob": 87, "Charlie": 92}

# Remove items
removed_score = scores.pop("Bob")  # Remove and return value
print(f"Removed Bob's score: {removed_score}")
print(f"Remaining scores: {scores}")

# Update with another dictionary
new_scores = {"Diana": 89, "Eve": 94}
scores.update(new_scores)
print(f"Updated scores: {scores}")

# Dictionary comprehension
squared_dict = {x: x**2 for x in range(1, 6)}
print(f"Squared numbers: {squared_dict}")

# Practical example: Word frequency counter
text = "python is great python is powerful python is fun"
word_count = {}
for word in text.split():
    word_count[word] = word_count.get(word, 0) + 1

print(f"\nWord frequencies: {word_count}")

3. Typecasting and User Input

Type Conversion

Convert between different data types when needed for calculations or operations.

# String to number conversion
age_str = "25"
age_num = int(age_str)
print(age_num + 5)          # 30

# Number to string conversion
score = 95
message = "Your score is " + str(score)

User Input

Get input from users to make your programs interactive and dynamic.

# Basic input (always returns string)

# Converting input to numbers
age = int(input("Enter your age: "))
height = float(input("Enter your height: "))

# Using input in calculations
current_age = int(input("Current age: "))
future_age = current_age + 10

# Multiple inputs at once
values = input("Enter two numbers: ").split()
num1, num2 = int(values[0]), int(values[1])
print(f"Sum: {num1 + num2}")

# Input validation example
while True:
    try:
        number = int(input("Enter a number: "))
        break
    except ValueError:
        print("Please enter a valid number!")

4. String Operations

String Basics

Strings are sequences of characters. Python provides many ways to work with text.

# Creating strings
name = "Python"
message = 'Learning is fun!'
multiline = """This is a
multiline string
example"""

# String indexing and slicing
print(name[0])          # 'P' (first character)
print(name[-1])         # 'n' (last character)
print(name[0:3])        # 'Pyt' (slice from 0 to 2)
print(name[2:])         # 'thon' (from index 2 to end)

# String length
print(len(name))        # 6

String Methods

Python provides many built-in methods to manipulate and analyze strings.

text = "  Hello World  "

# Case conversion
print(text.upper())         # "  HELLO WORLD  "
print(text.lower())         # "  hello world  "
print(text.title())         # "  Hello World  "
print(text.capitalize())    # "  hello world  "

# Whitespace removal
print(text.strip())         # "Hello World"
print(text.lstrip())        # "Hello World  "
print(text.rstrip())        # "  Hello World"

# String searching and replacing
print(text.find("World"))   # 8 (index of "World")
print(text.replace("World", "Python"))  # "  Hello Python  "
print(text.count("l"))      # 3 (count of 'l')

# String checking methods
email = "user@example.com"
print(email.startswith("user"))    # True
print(email.endswith(".com"))      # True
print("123".isdigit())             # True
print("abc".isalpha())             # True

String Formatting

Modern ways to format strings with variables and expressions.

name = "Alice"
age = 25
score = 95.5

# f-strings (recommended)
message = f"Hello {name}, you are {age} years old"
print(message)

# With expressions
print(f"Next year you'll be {age + 1}")
print(f"Your score: {score:.1f}%")

# .format() method
template = "Name: {}, Age: {}, Score: {:.2f}"
print(template.format(name, age, score))

# % formatting (older style)
print("Hello %s, you scored %.1f%%" % (name, score))

5. Conditional Statements

If-Else Statements

Make decisions in your code based on conditions. Control the flow of your program.

# Basic if-else
age = int(input("Enter your age: "))

if age >= 18:
    print("You can vote!")
else:
    print("You cannot vote yet.")

# Multiple conditions with elif
score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(f"Your grade is: {grade}")

# Nested conditions
weather = "sunny"
temperature = 25

if weather == "sunny":
    if temperature > 20:
        print("Perfect day for a picnic!")
    else:
        print("Sunny but a bit cold.")
else:
    print("Maybe stay indoors.")

Comparison Operators

Operators used to compare values and create conditions.

# Comparison operators
a, b = 10, 20

print(a == b)    # False (equal to)
print(a != b)    # True  (not equal to)
print(a < b)     # True  (less than)
print(a > b)     # False (greater than)
print(a <= b)    # True  (less than or equal)
print(a >= b)    # False (greater than or equal)

# Logical operators
x, y, z = 5, 10, 15

print(x < y and y < z)      # True (both conditions true)
print(x > y or y < z)       # True (at least one true)
print(not x > y)            # True (opposite of false)

# Membership operators
fruits = ["apple", "banana", "orange"]
print("apple" in fruits)        # True
print("grape" not in fruits)    # True

Advanced Conditionals

More complex conditional patterns and best practices.

# Ternary operator (conditional expression)
age = 20
status = "adult" if age >= 18 else "minor"
print(status)  # adult

# Multiple conditions
username = "admin"
password = "secret123"
is_logged_in = True

if username == "admin" and password == "secret123" and is_logged_in:
    print("Access granted")
else:
    print("Access denied")

# Checking multiple values
day = "Saturday"
if day in ["Saturday", "Sunday"]:
    print("It's weekend!")
else:
    print("It's a weekday.")

# Guard clauses (early returns in functions)
def process_age(age):
    if age < 0:
        return "Invalid age"
    if age < 13:
        return "Child"
    if age < 20:
        return "Teenager"
    return "Adult"

print(process_age(16))  # Teenager
